# 代码审查检查清单

## 概述

本文档提供 AI Agent 代码修改前后的检查清单，确保代码质量和架构合规性。

---

## 1. 修改前检查

### 1.1 文档阅读

- [ ] 已阅读架构文档 `docs/01-architecture/README.md`
- [ ] 已阅读相关模块文档（前端/后端）
- [ ] 已阅读编码规范 `docs/01-architecture/coding-standards.md`
- [ ] 已阅读 AI Agent 开发指南 `docs/01-architecture/ai-agent-guidelines.md`

### 1.2 架构范围确认

- [ ] 确认修改在现有架构设计范围内
- [ ] 不引入新的技术栈（除非明确批准）
- [ ] 不破坏现有模块边界
- [ ] 不违反设计原则

### 1.3 现有代码分析

- [ ] 已分析现有代码结构
- [ ] 确认是否可以复用现有组件/函数
- [ ] 确认是否需要新增逻辑（需用户确认）

---

## 2. 修改时检查

### 2.1 代码复用

- [ ] 是否复用了现有组件/函数？
- [ ] 是否优先修改而非新增？
- [ ] 是否避免了重复代码？

### 2.2 不可变数据原则

- [ ] 是否遵循不可变数据原则？
- [ ] React 状态更新是否使用函数式更新？
- [ ] 数组操作是否返回新数组？
- [ ] 对象更新是否使用展开运算符？

### 2.3 文件大小限制

#### 前端文件

| 文件类型 | 限制 | 检查 |
|----------|------|------|
| 页面组件 | 最大 400 行 | [ ] |
| UI 组件 | 最大 300 行 | [ ] |
| Hook 文件 | 最大 300 行 | [ ] |
| 工具函数 | 最大 200 行 | [ ] |

#### 后端文件

| 文件类型 | 限制 | 检查 |
|----------|------|------|
| Controller | 最大 300 行 | [ ] |
| Service | 最大 600 行 | [ ] |
| DTO | 最大 150 行 | [ ] |

### 2.4 函数大小限制

- [ ] 函数行数是否在 50 行以内？
- [ ] 嵌套层级是否不超过 4 层？
- [ ] 参数数量是否不超过 4 个？

### 2.5 新增逻辑确认

- [ ] 如果新增逻辑，是否已获得用户确认？
- [ ] 是否说明了新增原因？
- [ ] 是否说明了现有代码为何无法满足？

---

## 3. 修改后检查

### 3.1 静态检查

- [ ] 运行 ESLint 检查通过
- [ ] 运行 TypeScript 类型检查通过
- [ ] 无 console.log 调试代码
- [ ] 无 TODO/FIXME 遗留

### 3.2 测试检查

- [ ] 运行相关单元测试通过
- [ ] 运行相关集成测试通过
- [ ] 测试覆盖率是否达标（80%+）

### 3.3 代码质量

- [ ] 代码可读且命名清晰
- [ ] 注释是否必要且准确
- [ ] 无硬编码值（使用常量或配置）

### 3.4 安全检查

- [ ] 无硬编码的密钥或凭证
- [ ] 用户输入是否经过验证
- [ ] 敏感数据是否正确处理

---

## 4. 前端特定检查

### 4.1 组件选择

- [ ] 是否优先使用 shadcn/UI 组件？
- [ ] 是否复用了项目自定义组件？
- [ ] 如果新建自定义组件，是否已获得确认？

### 4.2 状态管理

- [ ] 服务端状态使用 TanStack Query？
- [ ] 客户端状态使用 Zustand？
- [ ] 协同状态使用 Yjs？

### 4.3 性能考虑

- [ ] 是否使用了动态导入（dynamic import）？
- [ ] 是否避免了不必要的重渲染？
- [ ] 是否使用了虚拟列表（大列表场景）？

---

## 5. 后端特定检查

### 5.1 NestJS 规范

- [ ] 模块结构是否符合规范？
- [ ] 是否使用依赖注入？
- [ ] 是否正确使用装饰器？

### 5.2 服务层

- [ ] 服务是否保持单一职责？
- [ ] 是否正确处理错误？
- [ ] 是否记录了必要的日志？

### 5.3 数据库操作

- [ ] 是否使用 Prisma 进行数据访问？
- [ ] 是否正确处理事务？
- [ ] 是否考虑了查询性能？

---

## 6. 协同功能检查

### 6.1 Yjs 相关

- [ ] 是否正确使用 Y.Doc？
- [ ] 是否正确处理 Provider 生命周期？
- [ ] 是否考虑了离线场景？

### 6.2 Awareness

- [ ] 是否正确更新 Awareness 状态？
- [ ] 是否正确监听远程状态变化？

---

## 7. 提交前最终确认

- [ ] 所有测试通过
- [ ] 代码格式正确
- [ ] 提交信息清晰
- [ ] 相关文档已更新

---

## 快速参考

### 核心规则

| 规则 | 限制 |
|------|------|
| 单文件最大行数 | 前端 400 行 / 后端 600 行 |
| 单函数最大行数 | 50 行 |
| 嵌套层级 | 4 层 |
| 参数数量 | 4 个 |
| 组件优先级 | shadcn/UI > 项目自定义组件 > 第三方 > 新建自定义组件 |
| 代码修改优先级 | 修改 > 删除 > 新增 |
| 新增逻辑 | **必须用户确认** |

### 常用命令

```bash
# 前端检查
cd collab-editor/frontend
pnpm lint
pnpm type-check
pnpm test

# 后端检查
cd collab-editor/backend
pnpm lint
pnpm type-check
pnpm test
```
